What is Power Query Editor?

Power Query Editor is a powerful data transformation and preparation tool integrated into Power BI Desktop. It provides a comprehensive environment for connecting to various data sources, shaping data, and transforming it before loading into your data model for analysis.

Key Capabilities

Power Query Editor allows you to connect to multiple data sources, perform over 50+ transformation operations, write custom M code, and create reusable query functions. All transformations are tracked as applied steps, making your data preparation process transparent and repeatable.

How to Access Power Query Editor: In Power BI Desktop, click on "Transform data" from the Home ribbon, or select "Edit Queries" to open the Power Query Editor interface.

Power Query Editor Interface

The Power Query Editor consists of four main components that work together to provide a comprehensive data transformation experience:

1. The Ribbon

Navigation experience with six main tabs (Home, Transform, Add Column, View, Tools, Help) providing access to all transformation functions and tools.

2. Queries Pane

Located on the left side, this pane displays all available queries in your workbook. You can organize queries into groups, rename them, and manage query dependencies.

3. Data Preview

The center pane displays a preview of your data with up to 1000 rows. You can switch between Data view, Schema view, and Diagram view to visualize your query structure.

4. Query Settings Pane

Located on the right side, showing the query name, properties, and all applied transformation steps. You can reorder, edit, or delete steps to modify your transformation logic.

Additional UI Elements

Status Bar: Located at the bottom of the window, displays query execution information, row counts, and profiling statistics.

Formula Bar: Shows the M code for the currently selected step, allowing direct code editing.

Home Tab - Core Operations

The Home tab contains the most frequently used operations for query management and basic data transformations. All features on this tab affect the entire table or query.

Close & Load Group

  • Close & Load: Applies all transformations and loads the data into Power BI Desktop data model
  • Close & Load To: Provides options to load data to specific locations (table, connection only, or both)

Data Sources Available:

  • Text/CSV: Import comma-separated or delimited text files
  • SQL Server: Connect to SQL Server databases
  • Web: Extract data from web pages and APIs
  • SharePoint: Connect to SharePoint lists and document libraries
  • OData Feed: Connect to OData services
  • Azure Services: Connect to Azure SQL Database, Azure Blob Storage, and more
  • Oracle, MySQL, PostgreSQL: Connect to various database systems
  • JSON, XML: Import structured data files
  • Folder: Combine multiple files from a folder
  • Blank Query: Start with an empty query to write custom M code

Recent Sources: Quick access to your recently used data connections

Query Management Group

  • Refresh Preview: Updates the data preview for the current query
  • Properties: Edit query name, description, and load settings
  • Advanced Editor: View and edit the complete M code for your query
  • Manage Queries: Delete, duplicate, or reference the current query
  • Delete: Remove the selected query from your workbook
  • Duplicate: Create an exact copy of the query
  • Reference: Create a new query that references the current one

  • Keep Rows: Keep top rows, bottom rows, range of rows, or duplicates
  • Remove Rows: Remove top rows, bottom rows, alternate rows, duplicates, or errors
  • Remove Duplicates: Keeps only unique rows based on all columns
  • Remove Errors: Removes rows containing errors in any column

Manage Columns Group

  • Choose Columns: Select specific columns to keep in your table
  • Remove Columns: Delete selected columns from the table
  • Remove Other Columns: Keep only the selected columns and remove all others
  • Go To Column: Navigate to a specific column by name

Transform Group (Basic)

  • Data Type: Change the data type of selected columns
  • Use First Row as Headers: Promote the first row to column headers
  • Detect Data Type: Automatically detect and apply appropriate data types
  • Replace Values: Find and replace specific values in selected columns
  • Replace Errors: Replace error values with a specified value
  • Fill: Fill down or fill up to propagate values

Combine Queries Group

  • Merge Queries: Combine two tables based on matching columns (similar to SQL JOIN)
  • Merge Queries as New: Create a new query from the merge operation
  • Append Queries: Stack tables vertically (similar to SQL UNION)

Merge Join Types Available

Left Outer, Right Outer, Full Outer, Inner, Left Anti, Right Anti

Structured Column Group

  • Expand: Expand nested tables, records, or lists into separate columns
  • Aggregate: Perform aggregations on nested tables (Sum, Count, Average, etc.)
  • Extract Values: Extract values from lists into text format

Note

These options are only active when you select columns containing nested structures (Table, List, or Record types). Regular data type columns will show these options as grayed out.

Transform Tab - Data Shaping Operations

The Transform tab provides advanced data transformation capabilities. A key difference from the Add Column tab: transformations here modify existing columns rather than creating new ones.

Table Operations

  • Group By: Aggregate data by grouping rows with common values

  • Transpose: Convert rows to columns and columns to rows
  • Reverse Rows: Reverse the order of all rows in the table
  • Count Rows: Returns the total number of rows as a single value
  • Use First Row as Headers: Promote the first data row to become column headers
  • Use Headers as First Row: Demote column headers to become the first data row

Column Operations

Pivot and Unpivot:

  • Pivot Column: Transform row values into column headers (wide format)
  • Unpivot Columns: Transform column headers into row values (long format)
  • Unpivot Other Columns: Unpivot all columns except the selected ones

Move and Rename:

Add Column Tab - Creating New Columns

The Add Column tab provides tools for creating new columns while preserving existing ones. This is the key difference from the Transform tab.

General Column Creation

Custom Column:

Create a new column using Power Query M formula language. Provides a dialog with:

  • New column name input field
  • Data type selection dropdown
  • Custom column formula editor with syntax highlighting
  • Available columns list for easy reference
  • Formula validation and error checking
Example Custom Column Formula: if [Sales] > 10000 then "High" else if [Sales] > 5000 then "Medium" else "Low"

Conditional Column:

Create IF-THEN-ELSE logic without writing code. The dialog provides:

  • New column name: Define the name of your new column
  • Column selection: Choose the column to evaluate
  • Operator selection: Equals, Not equals, Greater than, Less than, Greater than or equal to, Less than or equal to, Begins with, Ends with, Contains, Does not contain
  • Value input: Enter the value to compare against
  • Output value: Define what to return if condition is true
  • Add clause: Add multiple conditional clauses
  • Else clause: Define default output if no conditions match

Important Note

Conditional clauses are evaluated in order from top to bottom. The first condition that evaluates to true will return its output, and subsequent clauses will not be evaluated.

Index Column:

  • From 1: Create an index starting at 1
  • From 0: Create an index starting at 0
  • Custom: Define your own starting value and increment

Duplicate Column:

Create an exact copy of the selected column with a new name

Tools Tab - Diagnostics and Analysis

The Tools tab provides query diagnostics, profiling options, and tools for analyzing query performance and behavior.

Query Diagnostics

Query Diagnostics help you understand what the Power Query engine is doing behind the scenes, including:

  • Start Diagnostics: Begin recording diagnostic information
  • Stop Diagnostics: Stop recording and generate diagnostic report
  • Diagnose Step: Analyze performance of a specific transformation step

What Diagnostics Capture

Diagnostics capture query execution time, data source queries (SQL, API calls), query folding behavior, and resource consumption. The results are returned as new queries in your workbook showing detailed performance metrics.

Data Profiling Tools

  • Column Quality: Enable/disable column quality indicators
  • Column Distribution: Enable/disable distribution histograms
  • Column Profile: Enable/disable detailed column statistics
  • Data Profiling Scope: Choose between "First 1000 rows" or "Entire dataset"

Performance Consideration

Profiling the entire dataset can impact performance, especially with large data sources. Use "First 1000 rows" for faster preview during development.

AI Insights

  • Text Analytics: Analyze text for language detection, key phrase extraction, and sentiment analysis
  • Vision: Tag images, extract text from images (OCR)
  • Azure Machine Learning: Integrate with Azure ML models

Note: AI Insights require Azure Cognitive Services or Azure Machine Learning connections.

Always Allowed Data Sources

Manage which data sources are always allowed to refresh without security prompts. This is useful in enterprise environments with trusted data sources.

Advanced Features

M Formula Language

Power Query uses the M language (also known as Power Query Formula Language) for all transformations. While the UI generates M code automatically, you can write custom M code for advanced scenarios.

Key M Language Concepts:

  • Let-In Expressions: Standard structure for queries
  • Step References: Reference previous steps using their names
  • Functions: Create reusable custom functions
  • Parameters: Make queries dynamic with parameters
  • Error Handling: Use try...otherwise for robust queries
Example M Function: let GetCategory = (salesAmount as number) as text => if salesAmount > 10000 then "Premium" else if salesAmount > 5000 then "Standard" else "Basic", AddCategory = Table.AddColumn( PreviousStep, "Category", each GetCategory([Sales Amount]) ) in AddCategory

Query Folding

Query folding is one of the most important performance optimization features in Power Query. It pushes transformation logic back to the source database rather than processing data in Power Query.

Benefits of Query Folding:

  • Dramatically faster performance
  • Reduced memory consumption
  • Leverages source system's processing power
  • Reduces network data transfer

How to Check Query Folding:

Right-click on a step in Applied Steps and look for "View Native Query". If available, that step folds back to the source.

Operations That Support Folding:

Filter Rows
Remove Columns
Rename Columns
Select Columns
Change Data Types
Group By
Merge Queries (Joins)
Append Queries (Union)
Sort
Basic Calculations

Operations That Break Folding:

  • Adding custom columns with complex M functions
  • Text operations (split, extract, etc.)
  • Advanced transformations not supported by the source
  • Merging with non-foldable queries

Parameters

Parameters make queries dynamic by allowing values to be changed without editing the query code. Common uses include:

  • File Paths: Change source file locations
  • Date Ranges: Dynamic date filtering
  • Threshold Values: Adjustable business rules
  • Server Names: Switch between environments

Creating Parameters:

Home Tab Manage Parameters New Parameter

Define name, type, suggested values, default value, and current value.

Custom Functions

Create reusable functions to apply the same transformation logic across multiple queries or columns.

Creating a Custom Function:

  1. Create a query with sample data
  2. Apply desired transformations
  3. Right-click the query and select "Create Function"
  4. Define function parameters
  5. Use the function by invoking it on tables or values

Handling Nested Structures

Power Query can work with complex nested data structures from JSON, XML, and other sources.

Working with Tables:

  • Expand: Convert nested table into columns
  • Aggregate: Perform calculations on nested data

Working with Lists:

  • Extract Values: Convert list to delimited text
  • Expand to New Rows: Create a row for each list item
  • Aggregate: Count, sum, min, max on list values

Working with Records:

  • Expand Record: Convert record fields into columns
  • Extract Values: Get specific field values

Error Handling

Proper error handling ensures robust queries that don't fail when encountering unexpected data.

Error Handling Techniques:

  • Remove Errors: Filter out rows with errors
  • Replace Errors: Replace error values with defaults
  • Keep Errors: Isolate error rows for investigation
  • Try...Otherwise: M code for conditional error handling
Example Error Handling: = try Number.From([Value]) otherwise 0

Combining Files from Folder

A powerful feature for combining multiple files with the same structure into a single table.

How It Works:

  1. Connect to a folder containing files
  2. Power Query detects file types
  3. Click "Combine Files" button
  4. Define how to read the sample file
  5. Power Query creates a function and applies it to all files

Supported File Types

Excel workbooks, CSV files, JSON files, XML files, Text files, and more. All files must have the same structure.

Best Practices

Performance Optimization

  • Enable Query Folding: Perform transformations that fold back to the source
  • Filter Early: Remove unnecessary rows as early as possible
  • Remove Columns: Keep only columns you need
  • Avoid Custom Columns with Complex Logic: They can break query folding
  • Use Native Functions: Prefer built-in functions over custom M code
  • Disable Load for Intermediate Queries: Don't load helper queries to the model
  • Profile on Sample Data: Use "First 1000 rows" for profiling during development

Query Organization

  • Use Descriptive Names: Name queries clearly (e.g., "Sales_Cleaned" not "Query1")
  • Create Query Groups: Organize related queries into folders
  • Document Your Queries: Add descriptions in Query Properties
  • Use Query Dependencies: Reference queries instead of duplicating logic
  • Separate Staging and Final: Create staging queries for transformations, final queries for loading

Data Quality

  • Set Correct Data Types: Always define appropriate data types early
  • Handle Errors: Don't ignore errors; replace or remove them intentionally
  • Validate Data: Use column quality and distribution to spot issues
  • Trim Text: Remove extra spaces from text columns
  • Standardize Formats: Use consistent date, number, and text formats

Maintainability

  • Use Parameters: Make queries flexible with parameters instead of hardcoded values
  • Create Functions: Reuse logic across queries with custom functions
  • Comment M Code: Add comments in Advanced Editor for complex logic
  • Test Incrementally: Build queries step by step, testing each transformation
  • Version Control: Save backup copies before major changes

Security

  • Use Data Source Settings: Manage credentials properly
  • Privacy Levels: Set appropriate privacy levels for data sources
  • Avoid Hardcoded Credentials: Never embed passwords in queries
  • Review Data Sources: Regularly audit data source connections
